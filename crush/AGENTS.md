# Global Agent Preferences

## Git Conventions

- Commit only, never push unless asked.
- No hardcoded secrets in commits — use `$ENV_VAR` only.
- Scan staged files for secrets before committing.
- Commit messages: lowercase, imperative, no period (e.g. "add feature").
- Don't fix unrelated bugs or test failures (not your responsibility).
- Follow existing file naming and code style. Match what's already there.
